1 · The model — two layers, one rule

Typography in Primitiv is already solved at the variable level. Building web elements is mostly composition + colour, not inventing type scales.

LayerWhat it doesYou touch it when…
Context collection
4 modes: Dense · Compact · Comfortable · Spacious		 Holds every size decision: heading/*, body/*, label/*, display/*, overline/*, plus all spacing/anatomy. The active value is chosen by the frame's density mode override. Binding font size / line-height / spacing. Adding a new structural namespace (lists, tables…).
Intent collection
2 modes: Light · Dark		 Holds every colour decision: content/*, surface/*, border/*, action/*. Light/Dark inversion is automatic via the palette's own modes. Binding any fill, stroke, or text colour.

The two-collection rule

Context for size & spacing. Intent (content / surface / border / action) for colour. Bind to those and both density switching and light/dark theming work for free via frame mode overrides — no rebinding, no dark values to author.

The one gotcha that breaks everything — text styles silently kill density. A Figma TextStyle has no setExplicitVariableModeForCollection, so it always resolves at the collection's default mode (Compact) regardless of the frame's density override.
  • Loose document text (a heading dropped into a density-overridden page frame) → a text style is fine; it picks up the frame's density.
  • Inside a reusable component → never apply a text style. Bind font-family, font-style, font-size, line-height inline via node.setBoundVariable. If any of the four is missing from boundVariables, that node is unbound and won't respond to density.
Density is frame-owned. Never set an explicit Context mode override on a component variant — it locks instances to one density and breaks the consumer's frame-level switching.

5 · Structured content — components

The real build work. Each carries spacing/border/fill decisions beyond pure type, so each is a component with inline-bound typography. Per-element anatomy below.

Link Component Build

The only stateful inline element. An in-flow text link — distinct from the Button "link variant" (which is a button that looks like a link).

Parts

  • Label — text node. Inherits the surrounding body size (don't fix a size; bind to the same body/* as context, default body/md).
  • Underline — text-decoration (or a 1px bottom border for offset control).
  • Leading / trailing icon (optional) — e.g. external-link glyph; size to body cap-height.
  • Focus ring — reuse the framed-control 2-frame ring or an outline on focus.

Variants / properties

  • State (variant): default · hover · active · visited · focus · disabled
  • Tone (variant): default · muted (content/secondary) · inverse (on dark)
  • Icon (boolean/instance-swap): none · leading · trailing
  • Underline (boolean): always · hover-only

List + List Item Component Build list/*

Unordered & ordered. Build as two components: a List container (vertical auto-layout) holding List Item instances.

Parts

  • List — vertical auto-layout; item gap = list/item-gap; left padding = list/indent.
  • List Item — horizontal auto-layout: Marker + Content.
  • Marker — bullet glyph (disc/circle/square) or number/letter text; fill content/secondary.
  • Content — body text; may itself contain a nested List (recursion → indent compounds).

Variants / properties

  • Type: unordered · ordered
  • Marker style: disc · circle · square · decimal · lower-alpha · lower-roman
  • Item: leaf · has-nested · with-icon-marker (e.g. check for task lists)

Blockquote Component Done quote/*

2 × 2 × 5 = 20 variants: Tone × Citation × Size. Left accent bar = 3 px left stroke on the component frame (not a child frame — avoids FILL-in-HUG circular dependency). No italic: the bar + indentation are the quote signal.

Parts

  • Container — vertical auto-layout, FIXED 320 px wide, HUG height. Left stroke = accent bar.
  • Quote text — body/{size}, content/secondary. Both quote and citation bind to the same size slot.
  • Citation / footer (optional, Citation=with) — body/{size}, content/muted, end-aligned ("— Name").

Variants

  • Tone: default (neutral bar, border/strong) · accent (brand bar, border/focus)
  • Citation: with · without
  • Size: xs · sm · md · lg · xl → maps to body/{size} Context tokens

Rule of thumb for any element with no token:
  1. Type → reuse body/* or heading/* (never a new font scale).
  2. Colour → reuse content/* · surface/* · border/* · action/*.
  3. Spacing → if one space-* primitive fits, bind it directly. If it must scale with density, add a <element>/* namespace in Context, one alias per mode (the dropdown/* pattern).
  4. Only mint a brand-new primitive if no scale value fits — for typographic web elements, essentially never (the one such exception, monospace, is now landed: font-family.mono, §8).

8 · The monospace gap Resolved

Resolved. The system now has three faces — Khand (condensed display/label), Asta Sans (body) and JetBrains Mono (code). Every code-bearing element (inline code, code block, <kbd>, <samp>, <var>) binds to the new font-family.mono primitive.

  • font-family.mono = JetBrains Mono — chosen over Roboto Mono in an in-context comparison on the Inline code page. Variable added to Primitives (VariableID:601:9479); packages/tokens/src/primitives.json updated. Bind code nodes to it inline. See RFC 0012 D14.
  • Deferred to the emitter / new-typography session: the CSS monospace fallback stack (the variable holds a single family name for Figma binding, so the stack is emit-time — heading/text need sans-serif fallbacks too), and adding JetBrains Mono to the Google Fonts <link>s.

Resolved. Bold (<strong>): Asta Sans now ships Bold in Figma, so strong is a true in-family bold text style — the Khand-SemiBold borrow is no longer needed. Italic (<em>): no native italic, so em is a synthetic ~10° oblique built as a component (the skew is a node transform, not a text-style property). Both documented in their Figma descriptions.
