# `prose` — registry entry

The artefacts `primitiv add prose` resolves and copies into a consumer repo — the
opt-in **flow-rhythm** surface from RFC 0016. Unlike every other registry entry,
`prose` has **no headless `@primitiv-ui/react` counterpart**: `<Prose>` carries
zero behaviour (it only applies a class), so it ships entirely from the registry
(RFC 0016 D74). The load-bearing contract is the **`.primitiv-flow` class**; the
`<Prose>` wrapper is sugar over it.

## What it does

Inter-block spacing is **opt-in**. Bare prose carries no outer margins (RFC 0016
D66 zeroed them in the base layer). Drop `.primitiv-flow` (or `<Prose>`) on a
region and every **direct** child gets density-scoped vertical rhythm via a
one-directional owl (`> * + *`, `margin-block-start` only):

- **no margin collapse** and **no first/last-child bleed** — exactly one margin
  between two siblings, never above the first;
- **nothing leaks into a nested container** — a nested card/region opts in with
  its own `.primitiv-flow`;
- **variable rhythm** via role overrides — large air *above* headings, tight
  space *below* them (heading asymmetry); `h1`/`h2` pull `region`, `h3`/`h4` pull
  `section`.

The values are the density-scoped `--primitiv-flow-{tight,normal,section,region}`
tokens from the token layer, so a `[data-density]` ancestor re-rhythms the whole
subtree.

## Usage

```tsx
import { Prose } from "@/components/prose";

// Render a semantic element via asChild:
<Prose asChild>
  <article>
    <h1>Title</h1>
    <p>Body…</p>
    <h2>Section</h2>
    <p>More…</p>
  </article>
</Prose>

// Or the bare class on your own markup (styles-only consumers):
<article className="primitiv-flow">…</article>
```

Nest regions by wrapping each in its own `.primitiv-flow` — e.g. a card body
inside an article. Override a value locally by setting `--primitiv-flow-normal`
(etc.) on any ancestor; beat the rhythm entirely with an unlayered rule (no
`!important` needed — the owl sits in `primitiv.base` at low specificity).

## Files

| File | Authored? | Role |
|---|---|---|
| `contract.json` | **authored** | The styling contract — the `.primitiv-flow` root class and the `--primitiv-flow-*` rhythm tokens it consumes. |
| `styles.css` | **authored** | The canonical default theme: the owl + role overrides, in `@layer primitiv.base`. |
| `styles.scss` | **authored** | Byte-identical mirror of `styles.css` — `prose` owns no `--primitiv-prose-*` tokens, so there is no `$`-alias block to derive. |
| `prose.recipe.ts` | **authored** | A base-only `cva("primitiv-flow")` — the class as a recipe, for parity with the other entries. |
| `prose.tsx` | **authored** | The `<Prose>` wrapper. Hand-written (there is no primitive to generate from): a thin `asChild` (Slot) wrapper that renders a `<div>`, or the consumer's own element. |

Because there is no headless primitive, `prose.tsx`/`prose.recipe.ts` are **not**
generated by `primitiv-emit` and carry **no drift-guard test** (contrast the
generated wrappers, D53). `prose.tsx` is type-checked in CI by
`scripts/check-registry-types.mjs` like every other registry wrapper.

## Tokens

`prose` consumes the `flow/*` Context namespace (`--primitiv-flow-tight`,
`-normal`, `-section`, `-region`), density-scoped in the token layer. They are
shared, not component-owned, so they are present whenever the token layer is — no
per-component knob block. See RFC 0016 §3.
