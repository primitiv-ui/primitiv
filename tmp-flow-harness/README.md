# tmp-flow-harness — THROWAWAY visual check for RFC 0016 flow rhythm

Disposable Vite app to eyeball the spacing model before the values are baked into
`packages/tokens/src/context.json` and the `prose` registry component. **Delete
this whole directory once the flow values are signed off.** It lives outside the
pnpm workspace (`apps/*`, `packages/*`) on purpose, so it has its own isolated
`npm` install and never touches the wasm build.

## Run it

```sh
cd tmp-flow-harness
npm install
npm run dev      # open the printed http://localhost:5173
```

Left panel = **bare prose** (base element styles only — margins zeroed by RFC 0016
D66, so no inter-block spacing). Right panel = the same markup inside
**`.primitiv-flow`**. Toggle density / dark / margin-rings in the toolbar.

## What's prototyped here vs. shipped

- `public/tokens.css` + `public/primitiv-base.css` — generated by
  `cargo run -p primitiv-cli -- tokens --format css --out public/tokens.css`
  (the real emitter output; `primitiv-base.css` already reflects the margin revert).
- `public/flow.css` — **prototype** of the flow layer: seed `--primitiv-flow-*`
  values per density + the owl + role overrides. The owl/role CSS is destined for
  the `prose` registry component's `styles.css`; the per-density values for
  `context.json`. Tune the seeds here, then port.

## Seed flow values (per density → space-* alias)

| token | dense | compact | comfortable | spacious |
|-------|-------|---------|-------------|----------|
| `flow/tight`   | space-8  | space-10 | space-12 | space-16 |
| `flow/normal`  | space-12 | space-16 | space-20 | space-28 |
| `flow/section` | space-16 | space-24 | space-32 | space-40 |
| `flow/region`  | space-24 | space-32 | space-48 | space-56 |
